Phase 14 — Git, Code Review, Approvals & Development Orchestration

Status: Planned (Long-Term)
Priority: Critical
Estimated Duration: 10–18 development sessions

⸻

Objective

Make Relay capable of managing the complete software development lifecycle.

This phase introduces native Git operations, visual diffs, intelligent code review, pull request workflows, deployment orchestration, and approval pipelines that connect Relay Studio, Relay Mobile, and the Relay Agent into a unified engineering platform.

After this phase, Relay should be able to supervise an entire software project from planning through deployment while keeping a human in control of every important decision.

⸻

Product Philosophy

Git should never feel bolted onto Relay.

Repositories, branches, commits, reviews, approvals, and deployments should all be first-class concepts integrated into Relay’s task and agent systems.

The user should understand:

* What changed
* Why it changed
* Who (or which agent) changed it
* What happens next

⸻

Users should be able to:

* Browse repositories.

Design Notes

2026-07-15

Phase 14 completes Relay’s evolution into a full software engineering workflow platform. Rather than treating Git, code review, deployments, and approvals as separate tools, Relay unifies them into a single task-centric experience shared across Relay Studio, Relay Mobile, and the Relay Agent.

Every change should remain attributable, reviewable, reversible, and explainable. AI agents become collaborators in the development process, while humans retain authority over repository history, deployments, and production-impacting decisions.
